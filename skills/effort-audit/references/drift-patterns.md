# 精力漂移模式参考

Agent 在评估任务是否偏离方向时，可以参考以下常见模式。

## 高偏离信号

- 项目主题与个人方向完全无关。
- 用户开始为一个临时想法搭建完整基础设施。
- 大量时间花在非核心技能的打磨上。
- 项目的主要价值是"好玩"或"练手"，但已持续多天。

## 中度偏离信号

- 项目有关联，但用户开始深入非主攻领域。
- 原本简单的任务被扩展到超出原定范围。
- 用户在项目中反复优化不重要的细节。
- 项目占用时间明显超过最初估计。

## 可接受的偏离

- 明确的学习实验，且有截止时间。
- 为主目标服务的临时性辅助工作。
- 用户明确表达的有意探索。
- 短期的休息型项目，且不形成长期依赖。

## 判断原则

1. **看长期方向，不看短期情绪。**
2. **看实际投入，不看初始意图。**
3. **允许探索，但要求明确。**
4. **反复出现的偏离需要更新配置，而不是被忽视。**
